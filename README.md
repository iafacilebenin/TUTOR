## 🎯 What is Maître Étude?

Maître Étude is the **first AI-powered adaptive tutoring platform built natively for Beninese students** (Grade 1 through Terminal/Grade 12). Not an imported tool re-skinned for Africa — designed from scratch for the Beninese learner, aligned with the official national curriculum.

> Every child in Benin deserves an intelligent tutor available 24/7 — regardless of location, income, or school quality.

## ⚡ How It Works — The Adaptive Learning Engine

The core idea: **a student's learning path is never fixed**. Every interaction feeds back into the system, adjusting difficulty, pace, and content in real time.

```

       ▼
┌─────────────────────┐
│   Maître Étude      │
│   (React frontend)  │
└──────────┬──────────┘

           ▼
```

## 👥 Age-Group Adaptation — 8 to 18 Years

Maître Étude adapts content, vocabulary, examples, and pacing to four cognitive stages:

| Group | Age | School Level | Learning Style | Focus |
|-------|-----|-------------|----------------|-------|
| 🌱 **Éveil** | 8–10 | CP–CM2 | Play-based, visual, tactile | AI in daily life, image/sound recognition |
| 🔍 **Explorateur** | 11–13 | 6e–4e | Guided discovery, block logic | How algorithms work, creative projects |
| 🔨 **Bâtisseur** | 14–16 | 3e–2nde | Project-based, hands-on | Training simple models, building chatbots |
| 🚀 **Innovateur** | 17–18 | 1ère–Tle | Autonomous, deploy-oriented | Agents, NLP basics, public deployments |

Each group gets **different vocabulary, different examples, different depth** — same platform.

## 🔄 The Recursive Learning Cycle

Each learning session follows an 8-step cycle that **feeds itself**:

```
SESSION 1 — ANCHOR
  └─ Connect new concept to something the student already knows
     (Beninese context: markets, zémidjans, family, music)

SESSION 2 — DISCOVER
  └─ Guided demonstration. Student watches, asks, predicts.

SESSION 3 — EXPLORE
  └─ Student tries it. Makes mistakes. That's the data.

SESSION 4 — BUILD
  └─ First real project. Constraints given. Student chooses approach.

SESSION 5 — BREAK IT
  └─ Intentional challenge: edge cases, harder inputs, "what if?"

SESSION 6 — FIX IT
  └─ Debugging as learning. Claude explains WHY it broke.

SESSION 7 — PRESENT
  └─ Student explains their project to others.
     Teaching = deepest form of understanding.

SESSION 8 — INJECT
  └─ Session data analyzed → next cycle difficulty adjusted
     ┌──────────────────────────────────────────┐
     │  This is the recursive step.             │
     │  The system learns. The student learns.  │
     │  Both improve simultaneously.            │
     └──────────────────────────────────────────┘
```

**Result:** 8 sessions × N cycles = a compounding learning curve, not a flat one.
Every cycle starts smarter than the last.
